# JDBC Trap Questions

---

1. Can executeQuery run INSERT?
   No.

---

2. Does executeUpdate return ResultSet?
   No.

---

3. Is JDBC replaced by JPA?
   No. JPA uses JDBC internally.

---

4. Is batch processing parallel execution?
   No. It reduces network calls.

---

5. If connection closes, do statements close?
   Yes, but explicit closing is best practice.
